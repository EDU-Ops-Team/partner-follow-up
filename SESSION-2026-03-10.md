## Issues Encountered & Lessons Learned

### Airtable Shared View Scraping is Dead
**Issue:** The HTML scraping approach for Airtable stopped working — Airtable moved to fully client-side rendering with no access tokens embedded in the HTML.
**Resolution:** Switched to the official REST API with a Personal Access Token.
**Lesson:** Don't rely on reverse-engineered internal APIs. Use official APIs when available.

### Airtable Personal Access Token Permissions
**Issue:** Token created under our account returned 403 because the base was owned by another workspace.
**Resolution:** Duplicated the base into our own workspace so the token had access.
**Lesson:** Airtable Personal Access Tokens can only access bases in workspaces where the token creator has owner/creator access, even if the base is shared with them in the UI.

### Google Sheets API Not Enabled
**Issue:** `GOOGLE_SERVICE_ACCOUNT_KEY` was correct but Sheets API returned errors.
**Resolution:** Had to enable the Google Sheets API in the Google Cloud Console for the service account's project. Also fixed a truncated Sheet ID and a missing `!` in the `GOOGLE_SHEET_RANGE` env var (`Sheet1A:Z` → `Sheet1!A:Z`).
**Lesson:** Service account keys need the specific APIs enabled in their GCP project.

### Dev vs Prod Deployment Confusion
**Issue:** Spent significant time debugging because `CONVEX_DEPLOYMENT=prod:scintillating-schnauzer-916` env var override did NOT work with `npx convex run`. All our "prod" commands were actually hitting dev.
**Resolution:** Use `npx convex run --prod` flag instead of the env var override.
**Lesson:** Always use `--prod` flag for Convex CLI commands targeting production. The `CONVEX_DEPLOYMENT` env var override is unreliable.

### Vercel Build Caching
**Issue:** Dashboard showed stale data after changing `NEXT_PUBLIC_CONVEX_URL` in Vercel because the env var was set for Preview/Development but not Production.
**Resolution:** Set the env var for Production scope only, then triggered a fresh rebuild.
**Lesson:** `NEXT_PUBLIC_*` env vars are baked into the JS bundle at build time. Changes require a fresh build (not just a redeploy). Verify the Vercel env var scope (Production vs Preview vs Development).

### Airtable Column Name Ambiguity
**Issue:** The generic pattern `"date"` for `scheduledDate` matched both "Job Scheduled Date/Time" AND "Reporting Request Date/Time". The wrong column was being used as the scheduled date.
**Resolution:** Changed to exact column name patterns: `"job scheduled date"` for scheduled date, `"job status"` for status.
**Lesson:** When mapping external data columns, use the most specific pattern possible. Generic patterns like `"date"` or `"status"` are dangerous with wide tables.

### Address Matching False Positives
**Issue:** "835 Oak Creek Dr" fuzzy-matched to "995 Oak Creek Dr" because Levenshtein similarity was high (only 2 digits different).
**Resolution:** Added a street number validation guard — addresses with different leading numbers are rejected before fuzzy matching runs.
**Lesson:** Fuzzy string matching needs domain-specific guards. For addresses, the street number is a hard constraint that should never be fuzzy.

### Null Values in Convex Patch
**Issue:** Trying to clear optional fields by patching with `null` didn't work — Convex `patch` ignores null values.
**Resolution:** Set fields to empty strings (`""`) or `0` instead.
**Lesson:** Convex `db.patch()` with `null` does not unset optional fields. Use sentinel values or delete and recreate the document.
